Train-Test Split

Train-test split is used to divide the dataset into two parts: a training set and a testing set. The training set is used to train the machine learning model, while the testing set is used to evaluate how well the model performs on unseen data.

This helps prevent overfitting and provides a realistic estimate of model accuracy on new data.

Configure how the dataset is split into training and testing sets using the following parameters:

  • Test size: Proportion of the dataset used for testing (e.g., 0.2 = 20%).
  • Random state: Controls randomness for reproducible results.
  • Shuffle: Determines whether data is shuffled before splitting.

A common practice is to use 70–80% of the data for training and the remaining 20–30% for testing.
